Title: Hospital Appointment System

Description: A hospital appointment system manages the scheduling of medical consultations between patients and healthcare providers. It tracks doctor availability, records patient details, monitors the total number of consultations handled by the facility, and ensures that doctors are not overbooked. Patients and hospital staff use the system to ensure efficient hospital operations.
  
I have created a UML class diagram for the hospital appointment system, which includes the Doctor and Patient classes, their attributes, methods, and the relationships between them. The diagram illustrates how the classes interact, showing that patients can book and cancel appointments with doctors and that doctors have a set number of available slots.
